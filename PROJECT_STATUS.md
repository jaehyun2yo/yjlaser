작성일: 2026-06-17
전수조사 업데이트: 2026-06-19

## 요약

`yjlaser_website`는 유진레이저목형의 공개 홈페이지, 거래처 대시보드, 관리자 대시보드, 자체 웹하드, Worker/ERP 작업관리, NestJS 백엔드 API를 묶은 통합 웹 플랫폼이다. 회사 업무 흐름에서는 웹 문의 접수, 자체 웹하드 파일 관리, 작업관리, 납품 관리 축을 담당한다.

현재 진행률은 루트 `docs/roadmap.md` 기준 **11/17개 완료, 64.7%**이다. 운영 배포 기반은 갖춰져 있으나 `docs/task-board.md`와 `docs/release-readiness.md` 기준으로 Worker/mobile QA, 운영 env/migration 확인, 실시간 알림, 웹하드 UI/성능, API Rate Limiting이 다음 보완 영역이다.

## 핵심 기능

- 공개 홈페이지: 회사 소개, 포트폴리오, 블로그/공지, 견적 문의.
- 거래처 계정: 회원가입, 로그인, 거래처 대시보드, 주문/문의 현황, 웹하드 접근.
- 관리자 영역: 문의, 예약, 거래처, 포트폴리오, 게시글, 웹하드, 시스템 설정 관리.
- 자체 웹하드: Google Drive 기반 신규 파일/폴더 저장, PostgreSQL 메타데이터 기반 목록/권한/검색/휴지통, R2 레거시 파일 호환.
- Worker/ERP: 관리자 작업관리, 현장 모바일 작업자 화면, 작업 상태/공정 관리.
- API 서버: NestJS `/api/v1` 기반 웹하드/ERP API, Prisma/PostgreSQL, API Key/세션 인증.

## 연동

| 연동 대상                | 방식                                  | 목적                                   |
| ------------------------ | ------------------------------------- | -------------------------------------- |
| Supabase/PostgreSQL      | Prisma, 서버 API                      | 거래처, 웹하드, 주문, 작업 데이터 저장 |
| Google Drive             | Storage provider, upload/stream proxy | 신규 자체 웹하드 도면/첨부 파일 저장   |
| Cloudflare R2            | S3 호환 API, Presigned URL            | 포트폴리오 이미지와 기존 R2 참조 호환  |
| 외부웹하드동기화프로그램 | 자체 웹하드 API                       | LGU+ 웹하드 파일을 자체 웹하드로 수집  |
| Upstash Redis            | Rate limiting/cache 기반              | API 보호와 성능 보조                   |
| Vercel/Railway           | 배포 플랫폼                           | 프론트/백엔드 운영                     |
| Sentry                   | 모니터링                              | 웹 오류 추적                           |

## 연동 계약 점검

| 계약                                   | 현재 상태                          | 확인/보완 필요                                                 |
| -------------------------------------- | ---------------------------------- | -------------------------------------------------------------- |
| 외부웹하드동기화 -> 자체 웹하드 업로드 | NestJS API와 API Key 기반으로 사용 | 폴더 매핑 실패, stale ID, 대용량 파일 실패 시 복구 기준 문서화 |
| 거래처 문의 -> 웹하드 등록             | 부분 자동으로 문서화됨             | 문의 첨부가 자체 웹하드/주문 상태와 어디까지 연결되는지 확인   |
| Worker/ERP 상태 변경                   | 관리자/현장 화면에서 상태 관리     | 500 에러 재현 조건과 API/DB 경계 확인                          |
| 거래처 대시보드 알림                   | 계획/미완료                        | 알림 대상, 채널, 읽음 처리, 실패 시 재시도 기준 확정           |

## 부족한 점과 보완할 점

- Worker 작업관리: 작업 배정 자동화, 알림, 상태 변경 안정화가 필요하다.
- 거래처 대시보드: 주문 현황 실시간 알림과 셀프 주문 접수 기능이 미완료다.
- 웹하드: 폴더 트리 UI, 대용량 업로드 타임아웃, 파일 작업 UX를 더 다듬어야 한다.
- API 보호: Redis/Upstash 기반 Rate Limiting 강화가 필요하다.
- 데이터 접근 경계: Next.js API Routes와 NestJS API의 책임 범위를 최신 코드 기준으로 정리해야 한다.
