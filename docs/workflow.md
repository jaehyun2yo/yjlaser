# 업무 워크플로우

> 유진레이저목형의 도면 접수부터 납품까지 전체 업무 프로세스.
> 최종 업데이트: 2026-06-18
> 최신 운영 상태와 우선순위 판단은 `docs/company-project-overview.md`를 우선한다.

## 전체 흐름

```
접수 → 도면수정 → 컨펌 → DXF 가공 → 자동분류 → 네스팅 → 레이저가공 → 오시칼 → 납품
 (1)     (2)      (3)      (4)      (4-1)      (5)      (5-1)     (6)     (7)
```

---

## 1. 일 접수

거래처가 도면을 전달하는 경로는 2가지:

### 1-1. 웹사이트 문의
- 거래처가 `yjlaser.net`에서 문의 등록 (도면 첨부)
- 문의 등록 시 자체 웹하드에 자동 등록 (부분 구현 — 개선 검토 중)
- 담당: `yjlaser_website`

### 1-2. LGU+ 외부웹하드
- 거래처가 `only.webhard.co.kr`의 `올리기전용/{업체명}/` 폴더에 도면 업로드
- `외부웹하드동기화프로그램`이 자체 웹하드(Cloudflare R2)로 자동 동기화
- 동기화 상태: **문서상 운영 중, v1.5.15 설치 앱 장시간 운영 검증 필요**

**웹하드 폴더 구조:**
- `올리기전용/` — 거래처 → 유진레이저 (도면 전달)
- `내리기전용/` — 유진레이저 → 거래처 (수정도면/컨펌)

---

## 2. 도면 수정

- 접수된 도면을 도무송 규격에 맞게 CAD로 가공 (수동)

---

## 3. 수정 도면 전달 및 컨펌

- 수정 도면을 거래처에 전달 (외부웹하드 `내리기전용/` 또는 자체 웹하드, 거래처에 따라 다름)
- 거래처 확인 후 최종 도면을 다시 전달받음
- 수동 프로세스 (유선/메신저)

---

## 4. DXF 가공 및 작업폴더 이동

- 최종 도면을 CAD로 가공 → DXF로 저장
- 완성된 DXF를 네트워크 작업폴더에 이동

**작업폴더:** `\\192.168.0.6\home\데이터\유진MAIN\dxf\`

**파일명 규칙:** `{MMDD}-{순번} {업체명} {제품설명} {규격}.DXF`
- 예: `0219-1 샘플업체 [샘플제품] 목형의뢰 2절.DXF`
- 예: `0219-4 샘플거래처(12345).DXF`
- `00-`으로 시작하는 파일은 테스트/샘플

---

## 4-1. 관리프로그램 자동분류 + 청구서

- 파일명에서 업체명, 가격 자동 추출 → 업체별/월별 폴더로 자동 분류
- 청구서 자동 생성 (Excel) → 발송 (FAX/MMS/Email via Popbill API)
- 자동분류: **정상 동작**
- 청구서: 생성은 자동, 발송도 자동이지만 **전체적으로 업그레이드 필요**
- 담당: `유진레이저목형 관리프로그램`

---

## 5. 레이저 네스팅

- DXF 작업폴더의 새 파일을 감지 → 합판 배치 최적화 (네스팅)
- 현재 상태: **개선/리팩토링 진행 중** (UI 개선, 알고리즘 최적화)
- 담당: `레이저네스팅프로그램`

---

## 5-1. 레이저 가공

- 네스팅 결과 확인 후 '배치완료' → 네트워크 공유 Excel에 자동 저장
- 레이저가공기로 절단 작업 (수동)

---

## 6. 오시칼 작업

- 오시(접선)/칼(절단선) 수작업

---

## 7. 납품

- 직접 배달 또는 화물 발송
- `yjlaser_website`의 Worker 작업관리 시스템에서 관리

---

## Worker 작업관리 시스템

> 기존 ERP/칸반보드를 대체하는 새로운 시스템

- 사무실 작업, 현장 작업, 납품까지 통합 관리
- `yjlaser_website` 내에 구현
- 기존의 별도 납품관리 페이지도 Worker 시스템에 통합됨
