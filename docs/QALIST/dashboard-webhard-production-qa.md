# 관리자/업체 대시보드 및 웹하드 실제 QA 리스트

이 문서는 관리자 대시보드, 업체 대시보드, 웹하드, 문의 타임라인, 수정요청 업로드, 백업 프록시의 실제 브라우저/운영 환경 QA 체크리스트다.

자동 테스트로 커버된 항목도 세션, 쿠키, 권한, 업로드/다운로드, 외부 공유 흐름은 실제 환경에서 별도로 확인한다.

## 자동 테스트 확인 기록

- 2026-06-19 Codex 추가: 기본 Playwright E2E 수집 대상을 `ui-*.spec.ts`로 제한하고 `pnpm test:e2e:ui` 기준 UI 조작 Playwright E2E 묶음 추가. 로그인/업체등록/문의 첨부/관리자/업체/웹하드/작업자/납품/반응형 주요 흐름은 브라우저에서 실제 입력·클릭·파일 선택으로 검증한다.
- 2026-06-19 실제 실행: `pnpm test:e2e:ui -- --reporter=line`은 6 passed / 6 failed / 15 did not run. 실패 원인은 관리자 통합관리 로딩, 업체 반복 로그인 세션, 모바일 overflow, 웹하드 새 폴더 생성 반영, 작업자 대시보드 로딩, 업체등록 사업자등록증 Drive 업로드 500이다.
- 2026-06-19 수정 후 재실행: 업체등록 Drive folderKind 길이 초과, 로그인/hydration, 웹하드 새 폴더, 작업자 PIN 빠른 입력, 납품관리 탭 선택, 반복 실행용 업체등록 데이터 충돌을 수정했다. `pnpm test:e2e:ui -- --reporter=line` 최종 27 passed.
- 2026-05-19 Codex 확인: frontend 60개 suite / 508개 테스트, backend 24개 suite / 398개 테스트 통과
- 아래 체크 표시는 자동 테스트로 검증된 범위를 의미한다. 실제 운영 계정, 브라우저 쿠키, R2/NAS, 모바일 레이아웃, 배포 후 로그 확인이 필요한 항목은 미체크로 남긴다.
