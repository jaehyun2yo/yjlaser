# 위험 작업 게이트

YJLaser는 고객 도면, 거래처 정보, 외부웹하드, 자체 웹하드, 청구/발송,
프로덕션 DB/스토리지를 다룬다. 아래 작업은 fast path 금지이며 명시적 사용자
승인 전 실제 실행하지 않는다.

## 승인 필수 작업

| 영역 | 승인 필요한 작업 |
|---|---|
| Popbill | FAX/MMS/Email 실제 발송, 실제 과금/전송 API 호출 |
| Supabase | 프로덕션 DB 조회/수정, migration deploy, 데이터 삭제/백필 |
| Cloudflare R2 | 프로덕션 버킷 파일 업로드/삭제/이동, presigned URL 정책 변경 |
| Railway/Vercel | 프로덕션 배포, 환경변수 변경, 빌드/배포 설정 변경 |
| LGU+ 외부웹하드 | 실제 업로드/삭제/동기화, 거래처 폴더 구조 변경 |
| 고객 데이터 | 도면 파일, 거래처 연락처, 청구 정보, 작업 이력 열람/이동/삭제 |
| 인증/보안 | 관리자 인증, API key, 세션, 권한, public endpoint 변경 |
| 로컬/네트워크 파일 | `\\192.168.0.6\home\데이터\유진MAIN\dxf\` 실제 파일 변경 |

## 기본 처리 원칙

1. 먼저 dry-run, mock, staging, dev 환경으로 검증한다.
2. 실제 상태 변경 전에는 어떤 시스템에 어떤 영향이 있는지 말한다.
3. 사용자가 승인한 정확한 명령/행동만 실행한다.
4. 삭제, 배포, 마이그레이션은 rollback 또는 복구 경로를 확인한다.
5. 승인 없이 secret 값을 출력하거나 저장하지 않는다.

## 승인 요청 문구

```text
이 작업은 <system/data/users>에 실제 영향을 줄 수 있습니다.
먼저 <safe option>으로 확인할 수 있습니다.
실제 작업 `<exact command/action>`을 실행해도 될까요?
```

## 절대 하지 말 것

- 사용자 승인 없이 실제 FAX/MMS/Email 발송
- 사용자 승인 없이 프로덕션 DB migration deploy
- 사용자 승인 없이 고객 도면/거래처 정보 삭제 또는 외부 전송
- 사용자 승인 없이 `.env.local`, Doppler secret, API key 값을 응답에 노출
- 테스트 목적으로 프로덕션 외부웹하드/자체웹하드 파일을 변경
